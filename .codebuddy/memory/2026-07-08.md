## 修复重写分镜生成非法拼凑标签的 bug（严重）
现象：用户给重写需求"特写 外卖员 上楼脚步… 大爷 开门"后，AI 生成 `@R1_爷爷_外卖员`、`@R1_爷爷_敲门` 这类把标签和描述硬拼的非法标签（外卖员本应是独立新角色）。

根因：`/api/regenerate-prompt` 的 `shot` 分支**没把素材库标签清单传给 LLM**，LLM 只看到当前镜头的 materials 字段，遇到需求里的新角色"外卖员"不知用什么标签，就自创拼凑标签。

修复：
- server.ts `/api/regenerate-prompt` shot 分支：接收 `elements`(角色/场景/道具名)，注入"已有素材标签清单 + 下一个可用编号"，新增【素材标签铁律】禁止拼凑标签，并要求需求里出现的新角色/道具/场景分配下一个编号并上报到返回的 `newElements`；响应改为解析 JSON `{prompt, newElements}`（带纯文本兜底）。共用响应块仅修改了 regenerate 分支（用 "Regenerate Prompt API Error" 唯一锚定）。
- Detail.tsx `handleRegenerateShotPrompt`：请求体带上 `elements`；拿到响应后把 `newElements` 合并进素材库（按 name 去重）并一次性 `updateScript` 保存镜头提示词 + 新素材，提示新增个数。

注意：改了 server.ts，**必须重启 `npm run dev`**（tsx 不热重载后端）。改完前端也需刷新。

## 重写功能"不跟要求/自己加戏"二次修复（12:26）
用户反馈重写完全不按需求、还自行加戏（编出台词/情绪如"大爷微笑期待"）。
根因：用户需求被埋在旧镜头设定（含旧台词/情绪）之后，模型先被旧内容带偏；且"极致丰富电影级+融入微表情"指令推着自由发挥；temperature 0.8 太高。
二次修复（server.ts shot 分支）：
- 把 `userRequirements` 前置为【最高优先级指令】，并加"严格执行原则"：只写实化用户描述的景别+主体+动作，禁止加用户没写的剧情/台词/情绪/互动/角色；旧设定降级为仅格式参考。
- 去掉 action 字段里的"and 微表情"措辞。
- 规则1 增加"不是编剧、禁止新增需求外内容"；规则4 改为仅当用户明确写出台词才写，严禁编造情绪/内心戏。
- temperature 0.8 → 0.4（减少自由发挥）。

## 新增：分镜删除功能 + 分镜导入原封不动（12:45）
用户要求：1) 新增分镜删除功能；2) 分镜导入内容原封不动，只对 @xx 标签做格式化处理，其他不修改。
实现（src/pages/Detail.tsx，前端 Vite 热更新无需重启）：
- 删除：每个分镜卡片右上角（复制/编辑/重写按钮旁）新增红色 Trash2「删除分镜」按钮 → 弹出确认弹窗（deletingShotIndex 状态，仿删除集数弹窗）→ handleDeleteShot 按原始下标过滤并重新顺序编号 shotNumber。
- 导入原封不动：handleImportShots 现在 prompt 保留全文（含 **镜N** 标题行）；新增 normalizePromptTags（去掉 @ 后空格、修正连续 @、合并 @R1 _爷爷 / @R1_爷爷 _外卖员 这类"空格+下划线"断标签）与 extractMaterialTags（提取唯一 @R/@S/@P 标签并自动填入 materials 出场素材字段）。其余内容不改。
- 重要约定：导入分镜只把 @xx 填进 materials（显示用），**绝不会**自动写进 script.elements 素材库，避免把用户粘贴的拼凑标签（如 @R1_爷爷_外卖员）当新角色污染素材库（这是之前重写 bug 的根因）。
- 自测：用 Node 离线复刻导入/删除逻辑跑通（规范化、出场素材提取、原文保留、删除后重新编号全部通过）。
