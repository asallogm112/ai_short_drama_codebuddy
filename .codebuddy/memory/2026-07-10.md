## 分镜时间轴分段规则（用户明确需求，重要）
用户要求视频分镜的"时间轴分段"必须是**内容驱动、长短不一**的，不是均匀网格，也不是固定模板。
- ❌ 不要：0-0.5 / 0.5-1 / 1-1.5 均匀切；也不要 0-3 / 3-6 / 6-10 这种大段（画面感太差）。
- ✅ 要：如 0-2 / 2-2.5 / 2.5-5 / 5-6 / 6-8 / 8-8.5 / 8.5-10 这种按动作强度智能切分。
- **统一的是"规则"不是"数字"**：平缓交代/空镜 → 2-3秒长拍 + 远景/全景/侧拍；普通表演 → 1-2秒 + 中景；关键微动作/情绪转折 → 0.5-1秒短拍 + 近景/特写/低角度仰拍/俯拍。
- **每一段必须带：景别 + 拍摄角度 + 镜头运动 + 主体动作 + 光效/细节**。
- **相邻段姿态连续**（上段结尾=下段开头）。
- 具体起止秒数由 AI(DeepSeek) 按每个 shot 的内容现生成，不是套固定 schema。
- 素材头部格式保留：@R1_林晚 : @R1_林晚.jpg；结尾话术保留"直接生成视频…无版权无真人"。
- 目前仍在方案讨论阶段，未改代码。

## cleanElementPrompt 反复迭代修复素材提示词重复（server.ts）
用户多次反馈角色/素材提示词里 `只生成 1 张图片` 与 `图片比例 : ，图片比例 : 16:9` 重复、以及 `如果 生成过 , 就不要再生成了 . 切记 切记 , 因为要 保证 一致性  !` 废话。
修复历程：
1. 初版 `cleanElementPrompt`：去重「只生成 N 张图片」(genRe) + 调 `cleanImageRatio` 去重「图片比例」。初版已接入主生成(角色/场景/道具)与续写(原漏角色/道具分支，已补)。
2. 加 metaRe 删除元指令废话（如果生成过…/切记切记/保证一致性），连周围标点一起吞。
3. 多次发现逗号残留 bug：`cleanImageRatio` 删「图片比例 : 」会留前导逗号；`genRe` 删「只生成」会把两侧逗号挤成双逗号。最终在 `cleanElementPrompt` 末尾加连续逗号折叠 `replace(/[，,]{2,}/g,'，')` + 首尾清理。
最终实测（用用户真实脏串）：`只生成 1 张图片，图片比例 : ，图片比例 : 16:9，只生成 1 张图片` → `只生成 1 张图片，图片比例 : 16:9`；含元指令完整串 → 干净单逗号输出。lint 0 错误。
重要：重启命令被用户拒绝，当前运行进程(13:45)仍为旧版，需用户放行重启才能生效。旧 localStorage 已存脏数据需重新生成角色才更新。
